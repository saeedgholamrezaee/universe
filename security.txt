API Key:
 یک رشته‌ی متنی (معمولاً ترکیبی از حروف و اعداد) است.

این کلید توسط ارائه‌دهنده‌ی سرویس تولید می‌شود.

هر کلید معمولاً به یک حساب کاربری یا یک اپلیکیشن خاص اختصاص دارد.


Token based authentication
Token یک رشته‌ی رمزنگاری‌شده است که پس از ورود موفق کاربر (Login) توسط سرور صادر می‌شود.

این توکن معمولاً شامل اطلاعاتی مثل شناسه کاربر، زمان انقضا، و گاهی سطح دسترسی است.

رایج‌ترین فرمت‌ها: JWT (JSON Web Token) یا توکن‌های opaque (غیرقابل خواندن).


oAuth
کاربر می‌خواهد یک اپلیکیشن ثالث (مثلاً یک اپ موبایل) به داده‌هایش در یک سرویس دیگر (مثلاً Google Drive) دسترسی داشته باشد.

به جای دادن نام کاربری و رمز عبور، کاربر به اپلیکیشن اجازه می‌دهد از طریق OAuth یک Access Token دریافت کند.

این توکن فقط به همان اپلیکیشن و فقط برای دسترسی‌های مشخص‌شده اعتبار دارد.


Basic Authentication

ارسال نام کاربری و رمز عبور در هر درخواست (معمولاً در Header).

ساده اما ناامن اگر بدون HTTPS استفاده شود.


پس از ورود کاربر، سرور یک Session ID تولید می‌کند و در کوکی مرورگر ذخیره می‌شود.

در هر درخواست بعدی، مرورگر کوکی را ارسال می‌کند.

رایج در اپلیکیشن‌های سنتی (PHP، Django، Rails).


SAML (Security Assertion Markup Language)

استاندارد XML برای Single Sign-On (SSO).

بیشتر در سازمان‌ها و سیستم‌های Enterprise استفاده می‌شود.

کاربر یک بار وارد می‌شود و می‌تواند به چند سرویس مختلف دسترسی داشته باشد.


OpenID Connect (OIDC)

لایه‌ای روی OAuth 2.0 برای Authentication (در حالی که OAuth بیشتر برای Authorization است).

امکان ورود با حساب‌های گوگل، فیسبوک، مایکروسافت و غیره.

رایج در اپلیکیشن‌های مدرن.


Multi-Factor Authentication (MFA)

ترکیب چند عامل: رمز عبور + کد پیامکی + اپلیکیشن Authenticator.

امنیت بسیار بالاتر نسبت به روش‌های تک‌عاملی.


Biometric Authentication

استفاده از اثر انگشت، تشخیص چهره یا عنبیه.

بیشتر در موبایل و دستگاه‌های شخصی.


Certificate-Based Authentication

استفاده از گواهی دیجیتال (X.509 Certificates).

رایج در سیستم‌های سازمانی و ارتباطات امن (VPN، TLS Mutual Auth).
